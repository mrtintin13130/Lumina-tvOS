# Requirements: Lumina Native tvOS Client v1.2

**Defined:** 2026-06-08
**Core Value:** One authenticated Apple TV client can connect to a Lumina server, play HLS video through native AVKit on physical Apple TV, report progress, and resume reliably.

## v1.2 Requirements

Requirements for the stability, usability, and beta-hardening milestone. This milestone assumes the v1.1 backend contract and movie playback proof path exist, then makes the app reliable and comfortable enough for a small beta.

### Architecture And State

- [x] **ARCH-01**: Developer can reason about session, catalog, playback, diagnostics, and app routing without reading one central `AppModel` that owns every workflow.
- [x] **ARCH-02**: Existing setup, sign-in, Home, detail, playback, search, and settings behavior remains intact while responsibility is split into small, testable models or services.
- [x] **ARCH-03**: Async loads for catalog, search, detail, editorial sections, playback proof, and sign-out have clear stale-result/cancellation behavior and cannot mutate the wrong screen state.
- [x] **ARCH-04**: Security-sensitive state remains behind Keychain/token abstractions and cannot be logged, displayed, or stored in diagnostics through the refactor.

### tvOS User Experience

- [x] **UX-01**: User can navigate setup, server unavailable, sign-in, Home, grids, search, detail, editorial overlays, settings, and playback exit with predictable Siri Remote focus movement.
- [x] **UX-02**: Every screen has a sensible default focused element and restores prior focus where returning users expect it.
- [x] **UX-03**: User-visible placeholder actions are hidden, disabled with clear state, or implemented; the app does not present polished focused controls that only report "not wired yet."
- [x] **UX-04**: Setup and search reduce remote-text friction through discovery/manual fallback, clear URL normalization feedback, dictation-friendly fields, recent or saved server affordances, and safe empty/error states.
- [x] **UX-05**: Text, artwork, focus scale, shelves, overlays, and safe-area behavior remain readable and stable at living-room distance without layout jumps during async image loading.

### Playback Reliability

- [x] **PLAY-21**: Physical Apple TV playback proof covers movie start, pause, scrub where supported, exit, stop, relaunch, and resume against a live Lumina server.
- [x] **PLAY-22**: Progress and playback-session updates are sent at the expected cadence and on state transitions, and backend evidence confirms the saved position and stop/completion state.
- [x] **PLAY-23**: Expired stream token, HLS manifest/playlist/segment failure, missing media, server restart, and server unreachable scenarios produce user-safe recovery or terminal states.
- [x] **PLAY-24**: Audio and subtitle availability from backend tracks, HLS manifests, and AVKit media selection is recorded safely and verified on representative physical media where available.
- [x] **PLAY-25**: Playback exit and failure cleanup cannot leave stale playback sessions, stale `playbackProof`, or misleading Home/detail state.

### Diagnostics And Support

- [x] **DIAG-01**: User can open Settings and see safe support context: app version/build, tvOS version when available, server URL/validation state, server version/capability summary, signed-in user display name, and last safe error.
- [x] **DIAG-02**: Support diagnostics include backend-correlatable fields such as route key, HTTP status, correlation ID, playback session ID, media ID/kind, position, error category, and retryability when safe.
- [x] **DIAG-03**: Diagnostics redaction tests cover JWTs, stream tokens, Authorization headers, signed/tokenized URLs, passwords, local filesystem paths, SQL details, stack traces, and raw subprocess output.
- [x] **DIAG-04**: Diagnostics are support-only and do not introduce monetization analytics, unnecessary tracking, or third-party dependencies.

### Verification And Beta Readiness

- [x] **VER-11**: Generic tvOS build and build-for-testing commands pass with DerivedData outside user-local Xcode state.
- [x] **VER-12**: Unit tests cover the split architecture, state transitions, playback lifecycle cleanup, diagnostics support data, and security redaction.
- [x] **VER-13**: XCUITest replaces generated placeholders with smoke coverage for setup/sign-in, Home, search, detail, playback entry, Settings, and sign-out.
- [x] **VER-14**: Physical Apple TV QA matrix has safe evidence for playback, progress/resume, subtitles/audio where available, missing media, expired token, server restart/unreachable, long playback, and memory stability.
- [x] **VER-15**: TestFlight readiness notes identify reviewer path, demo server expectations, seed media coverage, privacy decisions, app icon/Top Shelf asset status, known deferrals, and remaining release blockers.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Large framework rewrite | v1.2 should reduce risk, not replace a working app with an overbuilt architecture. |
| Backend route rewrites | Backend changes should remain additive and focused on gaps found during proof. |
| QR/device pairing implementation | Valuable, but not required before stabilizing the current username/password MVP path. |
| Full genre/person/studio browsing | Placeholder links should not block beta if they are hidden or safely disabled. |
| Episode playback | Movie playback lifecycle and resume proof remain the beta gate. |
| Trailer playback | Trailer controls should be hidden/disabled unless implemented later. |
| Watchlist/favorite mutation UI | State display may remain, but mutation flows are deferred unless beta scope changes. |
| Top Shelf extension | Deferred until Home/playback quality is stable enough to deep-link confidently. |
| Monetization analytics or third-party telemetry | Diagnostics are support-only for this milestone. |
| App Store public launch | v1.2 prepares small beta readiness, not full public launch. |
