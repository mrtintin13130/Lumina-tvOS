# Structure

## Summary

The repository is a compact Xcode tvOS application scaffold plus a product requirements document. There are nine non-planning files outside `.git`, plus generated asset-catalog metadata. Source code is currently concentrated in three directories: `lumina/`, `luminaTests/`, and `luminaUITests/`.

## Top-Level Layout

```text
.
├── TVOS_CLIENT_PRD.md
├── lumina/
├── lumina.xcodeproj/
├── luminaTests/
├── luminaUITests/
└── scripts/
```

## Product Documentation

- `TVOS_CLIENT_PRD.md` contains the approved product requirements for the native tvOS client.
- The PRD references prior planning artifacts under `.planning/`, but those artifacts were not present before this map was created.
- The PRD is currently untracked in Git according to `git status --short`.

## App Source Directory

`lumina/` contains app source and assets:

- `lumina/luminaApp.swift`: `@main` app entry point.
- `lumina/ContentView.swift`: placeholder SwiftUI root view.
- `lumina/Assets.xcassets`: app asset catalog.
- `lumina/Preview Content/Preview Assets.xcassets`: SwiftUI preview assets.

## Asset Catalog Structure

Important asset paths:

- `lumina/Assets.xcassets/Contents.json`
- `lumina/Assets.xcassets/AccentColor.colorset/Contents.json`
- `lumina/Assets.xcassets/App Icon & Top Shelf Image.brandassets/Contents.json`
- `lumina/Assets.xcassets/App Icon & Top Shelf Image.brandassets/App Icon.imagestack`
- `lumina/Assets.xcassets/App Icon & Top Shelf Image.brandassets/App Icon - App Store.imagestack`
- `lumina/Assets.xcassets/App Icon & Top Shelf Image.brandassets/Top Shelf Image.imageset`
- `lumina/Assets.xcassets/App Icon & Top Shelf Image.brandassets/Top Shelf Image Wide.imageset`

The presence of Top Shelf image sets is generated project structure; Top Shelf product behavior is deferred in `TVOS_CLIENT_PRD.md`.

## Xcode Project

`lumina.xcodeproj/` contains:

- `lumina.xcodeproj/project.pbxproj`: target, build phase, build setting, and group definitions.
- `lumina.xcodeproj/project.xcworkspace/contents.xcworkspacedata`: workspace metadata.
- `lumina.xcodeproj/project.xcworkspace/xcshareddata/IDEWorkspaceChecks.plist`: shared Xcode workspace checks.
- `lumina.xcodeproj/xcuserdata/martin.xcuserdatad/xcschemes/xcschememanagement.plist`: user scheme management.
- `lumina.xcodeproj/project.xcworkspace/xcuserdata/martin.xcuserdatad/UserInterfaceState.xcuserstate`: local Xcode UI state.

The `.xcuserstate` file is user-local binary state and should generally not be committed if repository hygiene is tightened.

## Unit Tests

`luminaTests/` contains:

- `luminaTests/luminaTests.swift`: generated XCTestCase file with empty example and performance tests.

The file imports `@testable import lumina`.

## UI Tests

`luminaUITests/` contains:

- `luminaUITests/luminaUITests.swift`: generated UI-test scaffold that launches the app.
- `luminaUITests/luminaUITestsLaunchTests.swift`: generated launch screenshot test.

Both files use XCTest / XCUITest APIs.

## Scripts Directory

- `scripts/` exists but currently contains no files.

## Naming Conventions In Structure

- The project and target names are lowercase `lumina`.
- Swift type names currently include `luminaApp` and `ContentView`.
- Test bundle names follow Xcode defaults: `luminaTests` and `luminaUITests`.
- Asset names include spaces because they are generated by the tvOS asset catalog template, for example `App Icon & Top Shelf Image.brandassets`.

## Files To Watch During Early Development

- `lumina/ContentView.swift` is likely to be replaced or broken into feature screens soon.
- `lumina/luminaApp.swift` will likely become the composition point for app state and dependencies.
- `lumina.xcodeproj/project.pbxproj` must be updated whenever new Swift files are added through Xcode or project tooling.
- `TVOS_CLIENT_PRD.md` is the strongest current source of product and architecture intent.

## Suggested Future Organization

When implementation grows, consider adding directories under `lumina/` such as:

- `App/` for composition, routing, and app state.
- `Features/ServerSetup/`, `Features/Auth/`, `Features/Home/`, `Features/Catalog/`, `Features/Playback/`, and `Features/Settings/`.
- `Services/` for API clients, auth, playback, diagnostics, and storage.
- `Models/` for Codable DTOs and domain models.
- `Design/` for shared tvOS components and focus-aware styling.

This suggested structure is not present yet; it is a forward-compatible direction based on the PRD.
