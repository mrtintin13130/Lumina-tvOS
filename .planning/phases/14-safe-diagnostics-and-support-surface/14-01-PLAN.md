# 14-01 Plan: Settings Support Surface

## Goal

Settings shows useful support context without exposing secrets or turning into analytics.

## Steps

1. Add a support summary model derived from existing app/session/capability/diagnostic state.
2. Update `SettingsView` to render app/build, server validation, capability/API version, user display, last safe error, event count, and last support ID.
3. Keep actions focused and remote-friendly.

## Verification

- Settings compiles on tvOS.
- Support summary values are redacted and user-safe.
