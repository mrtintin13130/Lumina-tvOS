# 15-03 Plan - Physical Apple TV QA Matrix

## Goal

Create a safe physical QA checklist that makes the remaining proof gap explicit without exposing credentials, tokens, server internals, or media paths.

## Evidence Rules

- Evidence may include screenshots or short observations of user-facing screens.
- Evidence must not include passwords, JWTs, stream tokens, Authorization headers, signed media URLs, local filesystem paths, SQL details, stack traces, or raw subprocess output.
- Playback proof must use physical Apple TV hardware before MVP playback is considered complete.

## Acceptance

The matrix must list all beta-critical user journeys, current status, required evidence, and residual risk.
