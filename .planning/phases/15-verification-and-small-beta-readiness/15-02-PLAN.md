# 15-02 Plan - tvOS UI Smoke Tests

## Goal

Replace generated placeholder UI tests with compileable smoke tests that cover the expected beta-critical screens.

## Approach

- Add DEBUG-only seeded app launch states for Home, Detail, Search, Settings, and Sign In.
- Keep fixture launches from refreshing catalog data so tests do not depend on network or auth.
- Avoid unavailable touch APIs such as `XCUIElement.tap()` on tvOS.
- Assert visible screen surfaces and playback entry affordances.

## Acceptance

The UI test target must compile under generic `build-for-testing`, and the smoke test names must map to setup/sign-in, Home, search, detail/playback entry, Settings, and sign-out destination coverage.
