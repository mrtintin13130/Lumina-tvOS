# Phase 1 Plan 01: Define TV Client Route Matrix And Capability Contract

## Objective

- **What:** Define the route matrix and `GET /api/v1/system/capabilities` contract the tvOS app will consume.
- **Why:** The app shell and API client need explicit ownership, compatibility, and unsupported-feature behavior before implementation.
- **Output:** A developer-facing contract document plus supported/unsupported capability fixtures.
