# Quick Task: Catalog Horizontal Inset Root Cause

## Goal

Fix the remaining excessive spacing on Movies, TV Shows, and media detail pages by removing double horizontal safe-area insets while preserving Home's explicit 80 pt shelf alignment.

## Analysis

- Home is intentionally full-bleed and then applies `TVLayout.safeHorizontalPadding` to text, shelves, and status content.
- The non-Home catalog tabs were inside a safe-area-respecting `TabView` and also applied `TVLayout.safeHorizontalPadding`, creating a visual double inset.
- The detail page has the same horizontal issue outside the tab view: backdrop layers ignore safe area, but foreground content starts inside the root safe area and then adds explicit safe padding.

## Tasks

1. Let the shared catalog tab shell ignore horizontal safe area only, keeping top safe-area behavior for the native tvOS tab bar.
2. Let the detail page foreground container ignore horizontal safe area only, keeping top/bottom safe-area behavior unchanged.
3. Preserve `TVLayout.safeHorizontalPadding` as the single explicit content inset used by Home, grid pages, search, settings, and details.
4. Verify with static diff checks and a generic tvOS build.
